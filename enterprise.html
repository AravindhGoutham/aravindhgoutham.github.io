Enterprise Networks

In the Enterprise Networks course, I gained hands-on experience in designing and implementing scalable and secure network infrastructures. I completed a range of projects involving Spanning Tree Protocol (STP), VLAN configuration, and dynamic routing using RIP, EIGRP, and OSPF for both IPv4 and IPv6. The course also covered advanced topics such as Virtual Routing and Forwarding (VRF), Network Address Translation (NAT), and wireless LAN deployment. I implemented IPsec site-to-site VPNs to secure inter-site communications and worked with MPLS and RSVP-TE tunnels for efficient traffic engineering. These practical assignments deepened my understanding of enterprise-level networking and prepared me to handle complex real-world network environments.

Projects

  • STP (Spanning Tree Protocol): Configured STP to prevent switching loops in the network, ensuring a loop-free and redundant Layer 2 topology.
  • VLANs (Virtual LANs): Created and managed VLANs to logically segment the network into different broadcast domains, enhancing security, performance, and manageability.
  • EtherChannel: Implemented EtherChannel by bundling multiple physical links into a single logical connection to increase bandwidth and provide redundancy between switches or routers.
  • OSPF (Open Shortest Path First): Deployed OSPF as a link-state routing protocol to dynamically calculate the shortest path for data routing within an autonomous system using Dijkstra's algorithm.
  • MPLS-TE Tunnels: Configured MPLS Traffic Engineering tunnels using RSVP-TE to optimize path selection, improve bandwidth utilization, and provide predictable routing across service provider networks.
  • NAT: Configured static and dynamic NAT to allow internal devices to securely access external networks.
  • VRF: Isolated routing instances using Virtual Routing and Forwarding to simulate multi-tenant architecture.
  • IPv6 Tunnels: Set up IPv6-over-IPv4 tunnels to enable next-gen IP communication over legacy infrastructure.
  • IPsec Site-to-Site Tunnels: Established secure VPN tunnels between remote offices using IPsec protocols.
  • Stateful NAT: Enabled stateful NAT with session tracking for consistent IP translations and return path integrity.

Final Enterprise Networking Project

For the final practical in CSCI 5160: Introduction to Enterprise Networks at CU Boulder, I tackled a complex, hands-on project that emulated a real-world service provider environment using physical Cisco routers. This challenging lab required deep understanding and practical application of advanced networking concepts, including MPLS Traffic Engineering, IPv6 transition mechanisms, and GRE/IPsec site-to-site tunnels.

The network core was built with RSVP-based MPLS TE tunnels using Explicit Route Objects (EROs), bandwidth constraints, and LDP fallback for failover. I deployed GRE over IPsec tunnels between customer edge sites running OSPF, configured IPv6 GRE tunnels secured with IPsec, and enabled OSPFv3 for dynamic IPv6 routing. Dual-stack routing was implemented across the topology, supporting both IPv4 and IPv6 protocols.

Advanced services included IPv6 Prefix Delegation with /48 assignments to CE sites, where I configured DHCPv6 (stateful) and SLAAC (stateless) for host address assignment. I also integrated NAT-PT to bridge communication between IPv4-only and IPv6-only networks, and routed all external traffic through PE_99 using NAT and a custom public address pool. The project tested both theoretical knowledge and practical configuration skills, reinforcing real-world ISP and enterprise networking design.

Key Technologies:

  • MPLS, RSVP-TE, LDP
  • GRE, IPsec, OSPF, OSPFv3
  • NAT-PT, IPv6 Prefix Delegation
  • Stateful & Stateless DHCPv6, SLAAC
  • Dual-stack Routing, ECMP, Static Routing
