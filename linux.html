Linux System Administration

Collection of system-focused projects involving users, services, automation, and performance tuning.

Project Overview

This section showcases a series of Linux system administration projects carried out for a fictional enterprise, Dunder-Mifflin. The goal was to build a robust, automated, and secure server infrastructure across multiple operating systems and network segments.

Network Topology

The network design includes a Rocky Linux router, DNS and web servers across both Rocky and Debian distributions, a file server using AlmaLinux, and a FreeBSD host. Machines are divided between a LAN (10.21.32.0/24) and a DMZ (100.64.N.0/24) segment.

Features

  • User Management Automation: Automates account creation across Linux and FreeBSD machines including:
    • Username generation from full names
    • Consistent passwords and home directory setup
    • Group-based directory permissions
    • Sudo access for select users
    • System-wide umask configuration
  • Access Control & Privilege Management: Enforces role-based access:
    • SSH access for Pam, Kelly, Andy
    • Sudo-only access for Dwight (without group)
    • Timed shutdown access for Michael
    • Chown access for Jim (non-root only)
  • Multi-Site Apache Hosting: Configured on Machine C:
    • 3 virtual sites (Dunder Mifflin, Michael Scott Paper Co., Schrute Farms)
    • Custom DocumentRoots
    • User access mapping (Pam, Andy, Dwight)
    • Backup site mirror via rsync (Machine D)
    • Disk monitoring via authenticated CGI
  • Networking & Host Configuration: Static IPs and hostnames
  • DNS Server Setup: Forward/reverse zones via BIND
  • User & File Management: Ansible-driven NFS sharing
  • Security & Access Control: PAM rules, ACLs
  • Backup & Redundancy: RAID, rsync, and cron automation
  • System Automation: Role-based Ansible deployment
  • Time Sync: Chrony + NTP setup across all servers
