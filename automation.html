Network Automation

These projects demonstrate real-world applications of network automation by leveraging Python libraries like Netmiko and NAPALM, along with Ansible for configuration management. Device templates are dynamically generated using Jinja2, with continuous testing through Jenkins pipelines. Network performance and health are monitored using SNMP, Prometheus, and Grafana, enabling efficient, scalable, and reliable infrastructure management.

Tools & Technologies

Flask Web App

Developed a GUI-based Flask application that simplifies device configuration. It uses Netmiko and NAPALM libraries to automate the backend, replacing traditional CLI workflows.

Jinja2 Templating

Dynamic configurations are generated using Jinja2 templates based on device variables. This ensures consistency, eliminates manual errors, and makes deployments scalable.

Jenkins

Jenkins is used for continuous integration and delivery. Each GitHub push triggers tests and configuration jobs, improving automation reliability. Click to learn more →

Monitoring

Implemented SNMP-based monitoring with Prometheus and Grafana. This setup collects and visualizes device performance, resource usage, and network traffic in real time.

Troubleshooting

Scripts were built to fetch and compare current vs. previous configurations. This helps identify changes, improve visibility, and reduce troubleshooting time.

NETCONF

NETCONF enables remote configuration using a structured XML interface. It supports rollback, validation, and integrates cleanly into automated workflows.
