Network Automation

These projects demonstrate real-world applications of network automation by leveraging Python libraries like Netmiko and NAPALM, along with Ansible for configuration management. Device templates are dynamically generated using Jinja2, with continuous testing through Jenkins pipelines. Network performance and health are monitored using SNMP, Prometheus, and Grafana, enabling efficient, scalable, and reliable infrastructure management.

Flask Web App

Developed a GUI-based Flask application that simplifies device configurations. Uses Netmiko and NAPALM libraries for backend automation, replacing traditional CLI operations.

Jinja2 Templating

Dynamic device configs are generated using Jinja2 templates. Ensures consistency across environments, reducing manual error and improving deployment efficiency.

Jenkins

Integrated Jenkins for CI/CD. Every GitHub push triggers testing, version control, and automated deployments. Click to view detailed Jenkins pipeline setup →

Monitoring

Implemented SNMP-based monitoring with Prometheus and Grafana. Dashboards visualize performance metrics and traffic in real time.

Troubleshooting

Created automated scripts to fetch and diff configs across devices. Helps quickly detect anomalies and configuration drifts.

NETCONF

Used NETCONF to manage network devices over a secure XML-based interface, allowing configuration validation and rollback.
